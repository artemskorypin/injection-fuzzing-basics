# SQL Injection wordlists (`SQLi/SQL`)

Эта папка содержит словари нагрузок для SQL-инъекций в формате:

> **одна строка = одна нагрузка**

Фокус: **детект и фаззинг** (в том числе через ZAP), а не построение огромного набора эксплуатационных цепочек.  
Для глубокой эксплуатации предполагается использовать внешние ресурсы (например, PayloadsAllTheThings и т.п.).

---

## Общие принципы

- **Минимум шума:** без сотен почти одинаковых вариантов.
- **Фокус на идеях:** каждая строка представляет **отдельную технику** (boolean, time-based, WAF-bypass, stacked и т.д.).
- **DB-aware:** отдельные файлы под MSSQL / MySQL / Oracle / PostgreSQL.
- **Совместимость с ZAP:** файлы удобно использовать как:
  - *Custom payload lists*,
  - вход для *fuzzer’а*,
  - словари значений для параметров.

## Обзор файлов

### 1. Общие SQLi-словари

| Файл               | Назначение |
|--------------------|-----------|
| `sqli_general.txt` | Общие полезные строки для проверки SQL-инъекций в строковых параметрах. Boolean-логика (`or 1=1` и вариации), простые `UNION`, базовый WAF-обход (`/**/`), проверки на `information_schema` и т.п. Хороший «быстрый базовый» словарь. |
| `sqli_passive.txt` | Нагрузки, которые по возможности меньше ломают логику приложения, но позволяют увидеть аномальное поведение: сравнения строк, `LIKE`, `BETWEEN`, `IN`, нетипичные выражения, простые `OR 1=1`. Условно «мягкий» набор для пассивного тестирования. |
| `sqli_active.txt`  | Явно активные / пост-эксплуатационные payload’ы: создание пользователей, выдача прав, `xp_cmdshell`, просмотр файлов и системной информации. Это уже не детект, а «агрессивный» набор — используется только когда инъекция подтверждена и есть право так тестировать. |

---

### 2. MSSQL

| Файл                     | Назначение |
|--------------------------|-----------|
| `mssql_common.txt`       | Типичные MSSQL-паттерны: `@@version`, `sysobjects`, `sysdatabases`, `sysusers`, выборка логинов/таблиц, `UNION SELECT` с `@@version`, несколько полезных boolean-проверок на базе `ascii(...)` и т.п. Подходит для разведки структуры БД и проверки инъекций в контексте MSSQL. |
| `mssql_blind_waitfor.txt` | Blind / time-based payload’ы на `WAITFOR DELAY`. Включает простые тайминги (`WAITFOR DELAY '0:0:5'`), boolean-проверки через `IF (1=1)/(1=2)`, а также более сложные проверки `@@version`, `system_user`, `is_srvrolemember('sysadmin')`, `serverproperty(...)`. Используется, когда ошибки не видны и можно опираться только на задержки. |

---

### 3. MySQL

| Файл                          | Назначение |
|-------------------------------|-----------|
| `mysql_basic.txt`             | Базовые MySQL-нагрузки: классические `1 or 1=1`, `1' or '1'='1`, примитивные boolean-проверки и первые попытки `UNION`/ошибочных запросов. Подходит для начального фаззинга параметров. |
| `mysql_blind.txt`             | Blind + time-based payload’ы: `IF(условие,SLEEP(5),0)`, `SLEEP(5)`, `BENCHMARK(1000000,MD5(1))`, проверки через `isNULL(1/0)`, а также варианты с `ORDER BY ... SLEEP(5)`. Набор для ситуаций, когда ошибки не возвращаются или отфильтрованы, и остаётся только время отклика. |
| `mysql_waf_versioned_comments.txt` | MySQL-специфичный WAF-bypass через versioned comments (`/*!50000...*/`). Помогает обходить фильтры, которые режут ключевые слова, но пропускают MySQL-комментарии. Полезен при тестировании фильтров/WAF: позволяет понять, фильтруется ли `UNION/SELECT/OR` как слово или только конкретный паттерн. |

---

### 4. Oracle

| Файл         | Назначение |
|--------------|-----------|
| `oracle.txt` | Нагрузки для Oracle: классические `' or '1'='1`, вызовы `utl_http.request(...)` для проверки сетевых вызовов, time-based через `DBMS_LOCK.SLEEP` / `DBMS_SESSION.SLEEP`, выборка баннера (`v$version`), имени базы/хоста/пользователей (`ALL_USERS`, `USER$`, `ALL_TABLES`, `ALL_TAB_COLUMNS`, `DBA_ROLE_PRIVS`). Набор для разведки и blind-тестирования в Oracle-контексте. |

---

### 5. PostgreSQL

| Файл                 | Назначение |
|----------------------|-----------|
| `postgres_basic.txt` | Базовые PostgreSQL-payload’ы: `OR 1=1`, классические `' OR '1'='1--`, `UNION SELECT version()`, `current_database()`, `current_user`, выборка таблиц/колонок из `information_schema`, пользователей/ролей из `pg_user`/`pg_roles` и т.п. Для начальной проверки инъекций и разведки структуры. |
| `postgres_blind.txt` | Blind- и time-based-набор на базе `pg_sleep(5)`: boolean-проверки через `AND 1=(SELECT ...)`, `CASE WHEN ... THEN pg_sleep(5)`, а также небольшое количество общих SQLi-паттернов, адаптированных под PostgreSQL. Используется при скрытых/отфильтрованных ошибках. |

---

### 6. Вспомогательные / кросс-DB словари

Эти файлы — **строительные блоки**, а не самостоятельные эксплуатационные пайплайны.

| Файл                      | Назначение |
|---------------------------|-----------|
| `sql-dbms-fingerprint.txt` | Набор коротких выражений вида `func()=func()`, которые почти не влияют на логику приложения, но позволяют определить тип СУБД и факт выполнения функций. Подходит как «фингерпринт» движка (MySQL/MSSQL/Oracle/PostgreSQL/SQLite). |
| `sql-unicode-normalization.txt` | Небольшой набор URL-encoded Unicode-символов, которые в некоторых приложениях нормализуются до обычных `'` / `"`. Полезен при фаззинге багов на стыке Unicode-обработки и SQL-экранирования, особенно когда сырые `'` / `"` уже фильтруются. |
| `sql-polyglot.txt`         | Polyglot-паттерны, которые пытаются жить в нескольких контекстах сразу (строка, комментарий, SQL-выражение). Помогают проверять сложные фильтры и WAF’ы, где простые payload’ы уже режутся, а комбинированные — ещё нет. |
| `sql-alt-whitespace.txt`   | Набор альтернативных пробельных символов в URL-encoded виде (таб, переносы строк, NBSP и т.п.). Полезен для фаззинга парсеров и фильтров, которые по-разному обрабатывают whitespace. Обычно применяется в комбинации с другими словарями. |

---

### 7. Списки путей (Windows / Unix)

Эти два файла физически лежат рядом с SQL-словарами, но используются **во всём репозитории** там, где есть возможность читать файлы (SQLi, LFI, XXE, SSRF, command injection и др.).

| Файл            | Назначение |
|-----------------|-----------|
| `paths_win.txt` | Набор интересных путей на Windows: системные конфиги (`win.ini`, `system.ini`), hive’ы реестра (`SAM`, `SYSTEM`, `SOFTWARE`, backup в `repair`), логи (`WindowsUpdate.log`, eventlog’и), IIS/Apache/MySQL-конфиги, .NET/web-конфиги, пользовательские каталоги (`Desktop`, `Documents`, `Downloads`, `AppData`), SSH-ключи и прочие чувствительные файлы. |
| `paths_unix.txt` | Набор интересных путей на Unix-подобных системах: `/etc/passwd`, `/etc/shadow`, `sudoers`, `hosts`, логи (`auth.log`, `secure`, `syslog`, web-логи), конфиги Apache/Nginx/MySQL/PHP, web-корни (`/var/www/...`), `.env`, `.git`, bash-истории, SSH-ключи и cron’ы. |
