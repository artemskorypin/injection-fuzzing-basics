# XSS special

Данный каталог содержит дополнительные словари для фаззинга XSS, которые дополняют базовые словари из разделов Stored/Reflected и DOM-based XSS. Эти файлы предназначены для более широкого тестирования фильтров и обработчиков ввода и, как правило, используются после базовых словарей, в качестве «второй волны» проверки.

## Структура

- **XSS-Fuzzing.txt**  
  Широкий набор полезных нагрузок для фаззинга XSS.  
  Содержит различные варианты простых XSS-пейлоадов (в основном на основе `alert(1)` и аналогичных эффектов), оформленных под разные контексты: HTML-теги, атрибуты, обработчики событий, inline-скрипты и т.д.  
  Основная задача — помочь обнаружить нестандартные точки инъекции и особенности фильтрации, которые могут не проявиться при использовании минимального базового набора.

- **XSS-NO-PARENS.txt**  
  Нагрузки для XSS без использования круглых скобок `(` и `)`.  
  Полезны в случаях, когда фильтры или WAF блокируют скобки, но позволяют использовать альтернативные конструкции (например, через ссылки, обработчики событий, неявные вызовы и т.п.).  
  Словарь ориентирован на детект XSS в условиях ограниченной синтаксической выразительности.

## Назначение и ограничения

Словари из каталога `XSS_special`:

- предназначены **в первую очередь для детектирования XSS**, а не для построения полноценных эксплуатационных цепочек;
- не содержат нагрузок, нацеленных на кражу данных, долговременную фиксацию в браузере пользователя и другие агрессивные сценарии;
- могут создавать повышенную нагрузку и «шум» при тестировании, поэтому рекомендуются к использованию на тестовых стендах или в рамках согласованного тестирования безопасности.

Рекомендуемый порядок применения:

1. Сначала использовать компактные базовые словари (Stored/Reflected XSS, DOM-based XSS, `XSS-POLYGLOTS.txt`).
2. При необходимости углублённой проверки или исследования поведения фильтров — подключать словари из `XSS_special`.
